# Культурная лента

Минималистичный сайт-агрегатор по дипломному проекту: мировые новости, локальные культурные новости, конкурсы и подписка на e-mail-сводку.

## Стек

- Frontend: HTML, CSS, vanilla JavaScript.
- Backend foundation: Python 3, только стандартная библиотека.
- Хранение подписок для MVP: `data/subscriptions.json`.

Такой стек выбран как самый легкий для запуска и демонстрации: нет сборки, нет npm-зависимостей, сайт работает одной командой.

## Запуск

```bash
python backend/server.py
```

Откройте `http://localhost:3000`.

## Backend-основа

- `POST /api/subscriptions` сохраняет подписку.
- `GET /api/subscriptions` показывает сохраненные подписки.
- `GET /api/health` проверяет состояние API.
- `GET /api/content` возвращает последние распарсенные материалы.
- `POST /api/refresh` запускает внеочередное обновление источников.
- `backend/scheduler.py` содержит расчет следующей отправки и точку подключения очереди/SMTP.
- `backend/parser.py` парсит The Blueprint, BURO., Design Mate, Артгид, ART Узел, Vogue Business, Dezeen, designboom, e-flux и ArtConnect.
- `backend/content_store.py` запускает проверку обновления раз в 30 минут и выполняет новый парсинг, если данные старше 24 часов.

Для продакшена вместо JSON-файла можно подключить PostgreSQL, а `enqueueDigest` заменить на очередь задач и реальный SMTP-провайдер.

## Что может понадобиться дальше

Текущий парсер работает без внешних зависимостей и сохраняет результат в `data/content.json`. Если источник начнет отдавать контент только после выполнения JavaScript или включит жесткую антибот-защиту, нужно будет подключить Playwright-парсер для таких сайтов и хранить ключи/API-настройки в `.env`.
